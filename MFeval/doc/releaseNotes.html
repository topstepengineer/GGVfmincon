Release Notes

Home >

Toolbox uses the major.minor.patch version strategy. You can find out which version you have installed by typing ver('mfeval') from within MATLAB.

Version Release Date Notes
1.0.0 25-01-2017
  • First release of toolbox.
1.1.0 31-01-2017
  • Eqn (4.E40) Bt modified.
  • Allow a structure of parameters to be passed in instead of a TIR file name.
  • Replace for loops in the coeff checks with vector expressions.
  • Rho calculation reviewed avoiding (NaNs) in external effects.
  • Coefficient checks can be disabled.
  • Coefficients of friction are checked before being exported.
1.2.0 16-02-2017
  • Typo in the help text corrected.
  • Mux and Muy warnings can be deactivated with the coefficient checks.
  • Rolling resistance (4.E70), pneumatic trail (4.E73), Bt (4.E40), Dt (4.E43) and s(4.E76) equations have been corrected.
1.3.0 06-03-2017
  • Code modified for code generation included
  • Use mode implementation modified
  • Turn Slip equations corrected
  • Pneumatic trail equation fixed
1.4.0 03-07-2017
  • First implementation of the low speed mode
  • Publication of mfeval in MathWorks File Exchange
2.0.0 31-10-2018
  • Typos in the comments corrected
  • The following outputs have been included: b, Mzr, Cx, Cy, Cz, Kya, sigmax, sigmay and instKya
  • Improved input robustness check
  • Warnings and Errors IDs added
  • Negative deflections (rho) fixed
  • gamma_star definition only applied for usemode +20 to match MF-Tyre solver.
  • Code converted to object oriented
  • Simulink STI and CPI libraries have been included
  • Documentation updated to html
  • Code is validated using unitTesting prior release
  • Examples for each function and Simulink block included
2.0.1 31-10-2018
  • Fixed typo in the documentation
2.1.0 19-11-2018
  • Simulink Library updated to be more robust and consistent (STI interface depends on CPI interface)
  • All Simulink blocks and demos are saved in MATLAB 2017b
  • Advance user guide completed: including a demo showing how to fit a model
2.1.1 15-02-2019
  • Simulink Library updated to work with variable solvers.
  • Simulink memory blocks have been replaced with low pass filters with a pass band frequency of 15 Hz
  • coefficientCheck inputs have been updated to be compatible with MFit
2.2.0 03-06-2019
  • Road position implemented in STI
  • Road velocity implemented in STI
  • Some MATLAB Function blocks have been replaced with inbuilt Simulink blocks
  • The switch case to select the dynamics mode in CPI is done upstream to avoid unnecessary calculations
  • The passband frequency of the low pass filters has been increased to 100 Hz
  • Simulink library names updated. This will break the links from old models and needs to be updated
2.2.1 01-10-2019
  • Road friction can be passed as input in STI and CPI interfaces
  • Unnecessary zero crossing detections have been disabled in Simulink
  • Moved Fx and Fy filters to reside inside Switch dynamics mode (only used for non-linear)
  • In Mz calculation, Gyk_sub0 is now calculated after Fyo_sub0
2.3.0 23-10-2019
  • Slip stiffness (Kxk) is now output number 30
  • In STI vertical damping forces only actuate if the tire is in contact with the road
  • In STI the Slip Angle calculation has been updated to use the transformation matrix
3.0.0 09-12-2019
  • MF-Tyre 6.2 equations have been implemented. In 6.2 the loaded radius model is improved with camber dependencies
  • In STI and CPI muRoad has 2 components (longitudinal and lateral friction)
  • In STI and CPI the pressure is an optional input that can be now changed in the mask
  • In CPI the pressure is now saturated with the min and max pressure values of the .tir file
4.0.0 17-03-2020
  • MF-Tyre 5.2 equations have been implemented for backward compatibility. Note that in 5.2 the pressure has no effect
  • mfeval.readTIR can import Magic Formula 5.2 .tir files
  • Fixed typo in EHyp coefficient check warning message
  • Fixed psidot calculation in STI
  • Some variables in the code have been renamed to be more consistent with the book notation.
